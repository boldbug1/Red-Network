Installation & Deployment Manual

Ensure your network environment matches basic requirements: Podman or Docker (with Compose V2), Git, and an accessible terminal runner.

Admin Token Configuration

The administration console mapped to /-/admin is locked via a unique cryptographically secure adminToken string parsed directly from the environment initialization routine. Use internal system helpers to re-verify or change signatures. 

# Execute token update verification parameters inside Linux/macOS
./manage-token.sh

# Execute token update verification parameters inside Windows PowerShell environments
.\manage-token.ps1

After generating new tokens, ensure you call podman-compose restart red_engine to pass environment parameters directly into the Go system container variables.

Cryptographic Verification Protocols

R.E.D. calculates verification dynamically using Ed25519 signing operations over standard raw content strings against signatures stored inside isolated manifest.json parameters.

Example Node Vault Manifest Arrangement (manifest.json) 
{
  "relative/path/to/file.md": {
    "file_hash": "20bcee7014ad5cff...",
    "public_key": "66f32e250b0bafb2...",
    "signature": "61dcac5bd139dca0..."
  }
}
Request Loop Execution Model
  1. The engine tracks the active route, maps file location targets, and parses raw markdown text into local tracking chains.
  2. The engine calculates the SHA-256 hash signature block dynamically and compares tracking fields with the manifest keys.
  3. If hash matching values match cleanly, the Go runtime performs cross-verification of the Ed25519 payload array utilizing public keys stored inside repository root tracking tables (contributors.json).
  4. Passing metrics automatically generates an inline ✅ Verified Contributor badge while appending the absolute tracking hash into the template footers for manual operator configuration scans.
